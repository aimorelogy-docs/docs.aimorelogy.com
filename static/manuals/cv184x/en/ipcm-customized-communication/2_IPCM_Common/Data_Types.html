2.4.3. CVI_IPCM_CUST_KER_PORT_ST¶

[Description]

The starting port number available to kernel space on Linux; kernel-space registration is supported.

On Linux, messages whose port number is greater than or equal to CVI_IPCM_CUST_KER_PORT_ST are received by kernel space first and passed to the kernel-space callback.

The following cases apply:

  1. If both user space and kernel space register a callback, the AliOS msg first goes to kernel space.

After kernel-space processing, if the callback returns 1, user space does not receive the msg and its callback is not called.

After kernel-space processing, if the callback returns 0, the msg continues to user space and the user-space callback is also called.

  1. If kernel space registers a callback but user space does not, the msg is processed only in kernel space.

  2. If kernel space does not register a callback but user space does, the msg is processed only in user space.

  3. If neither space registers a callback, the msg is not processed and an error is printed.

[Definition]

#define CVI_IPCM_CUST_KER_PORT_ST 64

[Notes]

None

2.4.4. IPCM_MSG_TYPE_E¶

[Description]

Defines the msg category

[Definition]

typedef enum _IPCM_MSG_TYPE_E {
  IPCM_MSG_TYPE_SHM = 0,      // msg_param is share memory addr
  IPCM_MSG_TYPE_RAW_PARAM,    // msg_param is the param
} IPCM_MSG_TYPE_E;

[Members]

Member Name

Description

IPCM_MSG_TYPE_SHM

When the msg length exceeds 4 bytes, the callback returns IPCM_CUST_SHM_DATA_S (the shared-memory pool address and size).

IPCM_MSG_TYPE_RAW_PARAM

When the msg content does not exceed 4 bytes, the callback returns the msg param.

[Notes]

None

2.4.6. IPCM_CUST_MSG_S¶

[Description]

Defines the IPCM cust message structure. After a cust handler is registered, the handler returns a pointer to this structure when a cust message is received.

[Definition]

typedef struct _IPCM_CUST_MSG_S {
  CVI_U8 u8PortID;
  CVI_U8 u8MsgID : 7;
  CVI_U8 u8DataType : 1;
  union {
    IPCM_CUST_SHM_DATA_S stData; // while u8DataType is 0
    CVI_U32 u32Param; // while u8DataType is 1
  };
} IPCM_CUST_MSG_S;

[Members]

Member Name

Description

u8PortID;

port number

u8MsgID

msg number

u8DataType

msgData Types

0: return stData

1: return u32Param
