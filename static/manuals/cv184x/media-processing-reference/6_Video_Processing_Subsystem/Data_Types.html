6.4. 数据类型¶

VPSS 模块相关数据类型定义如下:

  • VPSS_DEVICE_NUM:定义 VPSS Device 数量。

  • VPSS_IP_NUM:定义 VPSS IP 数量。

  • VPSS_MAX_GRP_NUM:定义 VPSS GROUP 的最大个数。

  • VPSS_ONLINE_NUM:定义 VPSS ONLINE GROUP 的最大个数。

  • VPSS_MAX_CHN_NUM:定义 VPSS 通道的最大个数。

  • VPSS_MAX_PHY_CHN_NUM:定义 VPSS 物理通道的最大个数。

  • VPSS_MIN_IMAGE_WIDTH:定义 VPSS 图像的最小宽度。

  • VPSS_MIN_IMAGE_HEIGHT:定义 VPSS 图像的最小高度。

  • VPSS_MAX_IMAGE_WIDTH:定义 VPSS 图像的最大宽度。

  • VPSS_MAX_IMAGE_HEIGHT:定义 VPSS 图像的最大高度

  • VPSS0_HW_LIMIT_WIDTH:定义 VPSS0 硬件的限制宽度。

  • VPSS1_HW_LIMIT_WIDTH:定义 VPSS1 硬件的限制宽度。

  • VPSS2_HW_LIMIT_WIDTH:定义 VPSS2 硬件的限制宽度。

  • VPSS3_HW_LIMIT_WIDTH:定义 VPSS3 硬件的限制宽度。

  • VPSS_MAX_ZOOMIN:定义VPSS物理通道最大放大倍数。

  • VPSS_MAX_ZOOMOUT:定义 VPSS 物理通道的最大缩小倍数。

  • VPSS_RECT_NUM:定义 VPSS 物理通道画框最大个数。

  • VPSS_CHN0:定义 VPSS CHN 0。

  • VPSS_CHN1:定义 VPSS CHN 1。

  • VPSS_CHN2:定义 VPSS CHN 2。

  • VPSS_CHN3:定义 VPSS CHN 3。

  • VPSS_INVALID_CHN:定义 VPSS 非法通道号。

  • VPSS_INVALID_GRP:定义 VPSS 非法组号。

  • VPSS_GRP:定义 VPSS 组号。

  • VPSS_CHN:定义 VPSS 通道号。

  • VPSS_MODE_E:定义 VPSS 模式。

  • VPSS_INPUT_E:定义 VPSS 输入来源。

  • VPSS_MODE_S:定义VPSS工作模式结构体。

  • VPSS_ROUNDING_E:定义 Normalize时rounding的模式。

  • VPSS_CROP_COORDINATE_E:定义 CROP起点坐标模式。

  • VPSS_NORMALIZE_S:定义 Normalize功能所需信息。

  • VPSS_CROP_INFO_S:定义 CROP功能所需信息。

  • VPSS_GRP_ATTR_S:定义 VPSS GROUP属性。

  • VPSS_CHN_ATTR_S:定义 VPSS 物理通道属性。

  • VPSS_MOD_PARAM_S:通过接口设置模块参数。

  • PROC_AMP_E:定义 VPSS PROCAMP 类别。

  • PROC_AMP_CTRL_S:定义 VPSS PROCAMP 属性。

  • vpss_proc_amp_cfg:记录 ISP 绑定数据场景。

  • VPSS_ALL_PROC_AMP_S:定义 VPSS 所有组的颜色控制属性。

  • VPSS_LDC_ATTR_S:定义 VPSS 镜头畸变矫正结构体。

  • VPSS_CHN_BUF_WRAP_S: 定义VPSS Buffer卷绕属性。

  • VPSS_SCALE_COEF_E: 定义VPSS 图像缩放算法。

  • VPSS_RECT_S: 定义VPSS 边框属性。

  • VPSS_DRAW_RECT_S: 定义vpss画框信息。

  • VPSS_CONVERT_S: 定义vpss convert参数。

6.4.31. VPSS_NORMALIZE_S¶

【说明】

定义 Normalize功能所需信息。

【定义】

typedef struct _VPSS_NORMALIZE_S {
   CVI_BOOL bEnable;
   CVI_FLOAT factor[3];
   CVI_FLOAT mean[3];
   VPSS_ROUNDING_E rounding;
} VPSS_NORMALIZE_S;

【成员】

成员名称

描述

bEnable

Normalize使能开关。

factor

正规化因子。

1/8192 ~ 8191/8192

or

1/4096 ~ 8191/4096

mean

正规化方均根差。 0 ~ 255

rounding

对小数部分的处理方式。

【注意事项】

  1. 启用Normalize后的Output为int8, -128 ~ 127

  2. Normalize等效OpenCV中的convertTo()

cv::convertTo(image, CV_8S, factor, -mean);

  1. 根据VPSS CHN的format排列，结果会有所差异。
    若是PIXEL_FORMAT_RGB_888_PLANAR/PIXEL_FORMAT_RGB_888运作方式如下:
    (Pixel Value R) * factor[0] – mean[0]在接着做rounding。
    (Pixel Value G) * factor[1] – mean[1]在接着做rounding。
    (Pixel Value B) * factor[2] – mean[2]在接着做rounding。
    若是PIXEL_FORMAT_BGR_888运作方式如下:
    (Pixel Value B) * factor[0] – mean[0]在接着做rounding。
    (Pixel Value G) * factor[1] – mean[1]在接着做rounding。
    (Pixel Value R) * factor[2] – mean[2]在接着做rounding。

【相关数据类型及接口】

VPSS_ROUNDING_E

6.4.34. VPSS_CHN_ATTR_S¶

【说明】

定义 VPSS 物理通道属性。

【定义】

typedef struct _VPSS_CHN_ATTR_S {
   CVI_U32 u32Width;
   CVI_U32 u32Height;
   VIDEO_FORMAT_E enVideoFormat;
   PIXEL_FORMAT_E enPixelFormat;
   FRAME_RATE_CTRL_S stFrameRate;
   CVI_BOOL bMirror;
   CVI_BOOL bFlip;
   CVI_U32 u32Depth;
   ASPECT_RATIO_S stAspectRatio;
   VPSS_NORMALIZE_S stNormalize;
} VPSS_CHN_ATTR_S;

【成员】

成员名称

描述

u32Width

目标图像宽度

u32Height

目标图像高度

enVideoFormat

目标图像视频格式。

enPixelFormat

目标图像像素格式。

stFrameRate

帧率控制信息，当且仅当s32DstFrameRate小于s32SrcFrameRate时按比例控制帧率。 例如，若s32SrcFrameRate为60，s32DstFrameRate为30，则每隔一帧丢一帧。

bMirror

水平镜像使能。

bFlip

垂直翻转使能。

u32Depth

用户获取通道图像的队列长度。 静态属性。

stAspectRatio

幅形比参数。

stNormalize

Normalize以便加速后续TPU的运作。

【注意事项】

  • 输出图像宽度不能超过VPSS_MAX_IMAGE_WIDTH，高度不能超过VPSS_MAX_IMAGE_HEIGHT，全online不能放大。

  • 支持按比例控制帧率，-1表示不做帧率控制。

  • 假如开启幅形比，同时使能背景色，则不能在全online情况下使用。

  • 如果通道开启slice buffer，不支持Mirror和Flip，不支持帧率控制。

【相关数据类型及接口】

  • CVI_VPSS_SetChnAttr
