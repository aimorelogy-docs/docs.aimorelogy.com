4.4. 数据类型¶

视频输入相关数据类型定义如下，处理器差异的部分参见 各处理器支持能力 。

  • VI_MAX_DEV_NUM ：定义 VI 设备的最大个数。

  • VI_MAX_PHY_PIPE_NUM ：定义 VI 物理 PIPE 的最大个数。

  • VI_MAX_VIR_PIPE_NUM ：定义 VI 虚拟 PIPE 的最大个数。

  • VI_MAX_PIPE_NUM ：定义 VI PIPE 的最大个数。

  • VI_MAX_WDR_NUM ：定义 VI 支持最大WDR设备个数。

  • VI_MAX_VIR_CHN_NUM ：定义 VI 虚拟通道的最大个数。

  • VI_MAX_PHY_CHN_NUM ：定义 VI 物理通道的最大个数。

  • VI_MAX_EXT_CHN_NUM ：定义 VI 扩展通道的最大个数。

  • VI_EXT_CHN_START ：定义 VI 扩展通道起始通道号。

  • VI_MAX_EXTCHN_BIND_PER_CHN ：定义 VI 每个通道绑定扩展通道的最大个数。

  • VI_MAX_CHN_NUM ：定义 VI 物理信道和扩展信道的总个数。

  • VI_MAX_WDR_FRAME_NUM ：定义 VI 设备WDR模式下所支持的最大帧数。

  • VI_PIPE1_MAX_WIDTH ：VI PIPE1 处理图像的最大宽度。

  • VI_PIXEL_FORMAT ：VI 图像格式。

  • VI_VPSS_EXTRA_BUF ：定义 VI VPSS 额外缓冲区数量。

  • VI_CHN_0_BUF ：定义 VI 通道 0 缓冲区数量。

  • VI_CHN_1_BUF ：定义 VI 通道 1 缓冲区数量。

  • VI_CHN_2_BUF ：定义 VI 通道 2 缓冲区数量。

  • VI_CHN_3_BUF ：定义 VI 通道 3 缓冲区数量。

  • VI_BUF ：定义 VI 通道缓冲区数量。

  • VI_CMP_PARAM_SIZE ：定义 VI 比较参数大小。

  • VI_MAX_ADCHN_NUM ：定义 VI 最大模拟数字通道数。

  • VI_COMPMASK_NUM ：定义 VI 比较掩码数量。

  • VI_SHARPEN_GAIN_NUM ：定义 VI 锐化增益参数数量。

  • VI_AUTO_ISO_STRENGTH_NUM ：定义 VI 自动ISO强度参数数量。

  • VI_INVALID_CHN ：定义 VI 无效的视频输入通道编号。

  • VI_DATA_TYPE_E ：VI 输入数据类型枚举。

  • VI_DEV_ATTR_S ：定义视频输入设备的属性。

  • VI_DEV_BIND_PIPE_S ：定义 VI DEV 与 PIPE 的绑定关系。

  • VI_PIPE_ATTR_S ：定义 VI PIPE 属性。

  • VI_DUMP_TYPE_E ：枚举 dump 类型。

  • VI_DUMP_ATTR_S ：定义 VI PIPE dump 属性。

  • VI_CHN_ATTR_S ：定义 VI 通道属性。

  • VI_CROP_INFO_S ：定义 VI CROP 信息结构体。

  • VI_DEV_TIMING_ATTR_S ：自产生时序属性。

  • VI_PIPE_STATUS_S ：定义 VI PIPE 状态信息。

  • VI_CHN_STATUS_S ：定义 VI 通道状态信息。

  • VI_PIPE_FRAME_SOURCE_E ：定义 VI PIPE 数据的来源类型。

  • VI_LDC_ATTR_S ：定义VI镜头畸变矫正结构体。

  • VI_STATE_E ：枚举 VI 状态信息。

  • VI_SYNC_CFG_S ：BT.601或DC定时的同步信息。

  • VI_BT656_SYNC_CFG_S ：BT.656的同步信息。

  • VI_PIPE_BYPASS_MODE_E ：VI pipeline bypass模式。

  • VI_CROP_COORDINATE_E ：裁剪坐标的不同类型。

  • VI_MOD_PARAM_S ：VI模块参数。

  • VI_EARLY_INTERRUPT_S ：提前中断的参数。

  • MLSC_GAIN_LUT_S ：增益查找表。

  • VI_DUMP_REGISTER_TABLE_S ：寄存器转存数据。

  • VI_PM_OPS_S ：电源管理相关操作。

  • VI_SMOOTH_RAW_DUMP_INFO_S ：连续raw图转存信息。

  • VI_INTF_MODE_E ：视频输入接口模式。

  • VI_INPUT_MODE_E ：输入模式。

  • VI_WORK_MODE_E ：工作模式。

  • VI_SCAN_MODE_E ：扫描模式。

  • VI_YUV_DATA_SEQ_E ：YUV数据序列。

  • VI_CLK_EDGE_E ：时钟边缘模式。

  • VI_COMPONENT_MODE_E ：组件模式。

  • VI_COMBINE_MODE_E ：亮度色度信号组合模式。

  • VI_VSYNC_E ：垂直同步信号的属性。

  • VI_VSYNC_NEG_E ：垂直同步信号的极性。

  • VI_HSYNC_E ：水平同步信号的属性。

  • VI_HSYNC_NEG_E ：水平同步信号的极性。

  • VI_VSYNC_VALID_E ：有效垂直同步信号的属性。

  • VI_VSYNC_VALID_NEG_E ：有效垂直同步信号的极性。

  • VI_WDR_ATTR_S ：WDR属性。

  • VI_TIMING_BLANK_S ：输入时序的消隐信息。

  • VI_BT656_FIXCODE_E ：BT.656定时参考码的最高位。

  • VI_BT656_FIELD_POLAR_E ：BT.656定时参考码的字段指示符位（F）的极性。

  • VI_DEV_ATTR_EX_S ：VI设备的扩展属性。

  • VI_EXT_CHN_SOURCE_E ：VI扩展通道数据源类型。

  • VI_EXT_CHN_ATTR_S ：VI扩展通道属性结构体。

  • VI_USERPIC_BGC_S：结构体已弃用。

  • VI_LOW_DELAY_INFO_S：结构体已弃用。

  • VI_CMP_PARAM_S：结构体已弃用。

  • VI_USERPIC_ATTR_S：结构体已弃用。

  • VI_PIPE_SHARPEN_ATTR_S：结构体已弃用。

  • VI_PIPE_REPEAT_MODE_E：结构体已弃用。

  • VI_NR_VERSION_E：结构体已弃用。

  • NRX_PARAM_MANUAL_V1_S：结构体已弃用。

  • NRX_PARAM_AUTO_V1_S：结构体已弃用。

  • tV500_VI_SFy：结构体已弃用。

  • VI_PIPE_NRX_PARAM_V2_S：结构体已弃用。

  • NRX_PARAM_AUTO_V2_S：结构体已弃用。

  • NRX_PARAM_V2_S：结构体已弃用。

  • VI_PIPE_NRX_PARAM_S：结构体已弃用。

  • VI_RAW_INFO_S：结构体已弃用。

4.4.28. VI_DEV_ATTR_S¶

【说明】

定义VI设备的属性。

【定义】

typedef struct _VI_DEV_ATTR_S {
   VI_INTF_MODE_E enIntfMode;
   VI_WORK_MODE_E enWorkMode;
   VI_SCAN_MODE_E enScanMode;
   CVI_S32 as32AdChnId[VI_MAX_ADCHN_NUM];
   VI_YUV_DATA_SEQ_E enDataSeq;
   VI_SYNC_CFG_S stSynCfg;
   VI_DATA_TYPE_E enInputDataType;
   SIZE_S stSize;
   VI_WDR_ATTR_S stWDRAttr;
   BAYER_FORMAT_E enBayerFormat;
   CVI_U32 chn_num;
   CVI_U32 snrFps;
   CVI_U64 phy_addr;
   CVI_U32 phy_size;
   CVI_BOOL isMux;
   CVI_U8 switchGpioIdx;
   CVI_U8 switchGpioPin;
   CVI_U8 switchGPioPol;
   CVI_BOOL disEnableSbm;
   CVI_BOOL isFrmCtrl;
   CVI_U8 dstFrm;
} VI_DEV_ATTR_S;

【成员】

成员名称

描述

enIntfMode

Sensor接口模式。

enWorkMode

1、2、4 路复合工作模式。

enScanMode

输入扫描模式 (逐行、隔行)。

as32AdChnId[VI_MAX_ADCHN_NUM]

取值范围[-1,3]，推荐统一设置为默认值-1，此参数无意义。

enDataSeq

输入数据顺序 (仅支持 yuv格式)。

stSynCfg

同步时序配置，BT.601模式时必须配置，其它模式时无效。

enInputDataType

输入数据类型，Sensor输入一般为 RGB，AD 输入一般为 YUV。

stSize

VI 设备可设置要捕获图像的高宽，捕获图像的最小宽高与最大宽高：

宽度：[VI_DEV_MIN_WIDTH，VI_DEV_MAX_WIDTH]。

高度：[VI_DEV_MIN_HEIGHT，VI_DEV_MAX_HEIGHT]。

stWDRAttr

WDR 属性。

enBayerFormat

设备的bayer format，当inputDataType为RGB必须设置此属性。

chn_num

设备的通道数。

snrFps

设备的初始帧率。

phy_addr

仅ALIOS/DUAL OS支持, 若不为0, VI使用用户分配的物理地址,使用前需保证物理地址的合法性.

phy_size

仅ALIOS/DUAL OS支持, 所使用物理地址内存大小.

isMux

是否多sensor使用同一个dev。

switchGpioIdx

仅ALIOS/DUAL OS支持。gpio bank,-1表示不使用gpio。

switchGpioPin

gpio引脚，-1表示不使用gpio。

switchGPioPol

gpio值[0,1]，-1表示不使用gpio

disEnableSbm

传感器定时是否支持SBM。

isFrmCtrl

仅ALIOS/DUAL OS支持。是否启用帧控制

dstFrm

仅ALIOS/DUAL OS支持。switch设置目标帧数

【注意事项】
