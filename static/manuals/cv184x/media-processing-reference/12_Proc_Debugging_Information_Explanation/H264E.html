12.6. H264E¶

【调试信息】

# cat /proc/cvitek/h264e
Module: [H264E] System Build Time [#1 SMP PREEMPT Sat Jan 23 02:50:26 CST 2021]
-----MODULE PARAM-------------------------------------------------
OnePack: 0       H264eVBSource: 0        PowerSaveEn: 0  MiniBufMode: 0  bQpHstgrmEn: 0
-----CHN ATTR-----------------------------------------------------
ID: 0    MaxWidth: 1920  MaxHeight: 1080         Width: 1920     Height: 1080    Profile: 0      C2GEn: 0        BufSize: 0      ByFrame: 1      GopMode: NORMALP        MaxStrCnt: 0

【调试信息分析】

记录当前H.264视频编码属性配置以及状态信息。

【参数说明】

参数

描述

MODULEPARAM

OnePack

获取流的模式

0: 以多包模式获得流。

1: 以单包模式获得流。

H264eVBSource

为参考帧和重建帧获取VB的模式

2: 使用私有VB

3: VB由用户分配

PowerSaveEn

低功耗参数控制开关

0: 低功耗参数被禁用。

1: 低功耗参数被启用。

MiniBufMode

分配码流缓冲区的模式

0: 基于分辨率分配码流缓冲区。

1: 流缓冲区大小的下限是32 KB。

用户需要确保分配的流缓冲区大小合适。

bQpHstgrmEn

是否在高级流信息中显示QP直方图

0: no

1: yes

CHN ATTR

ID

通道号

MaxWidth

编码通道的最大宽度 (unit: pixel)

MaxHeight

编码通道的最大高度 (unit: pixel)

Width

宽度 (unit: pixel)

Height

高度 (unit: pixel)

profile

编码通道 profile 级别

Base: baseline

MP: main profile

HP: high profile

C2GEn

彩色转灰色使能

Value range: {0, 1}

BufSize

码流缓冲区大小 (unit: byte)

ByFrame

是否按帧获取流

Value range: {0, 1}

GopMode

GOP 模式

MaxStrCnt

流缓冲区中的最大帧数

默认值: 200
