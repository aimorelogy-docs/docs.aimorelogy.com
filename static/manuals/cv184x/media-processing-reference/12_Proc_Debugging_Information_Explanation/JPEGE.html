12.7. JPEGE¶

【调试信息】

# cat /proc/cvitek/jpege
Module: [JPEGE] System Build Time [#1 SMP PREEMPT Sat Jan 23 02:50:26 CST 2021]
-----MODULE PARAM-------------------------------------------------
OnePack: 0       JpegeMiniBufMode: 0     JpegClearStreamBuf: 0   JpegeDeringMode: 0
-----CHN ATTR-----------------------------------------------------
ID: 0    bMjpeg: Y       PicType: YUV422         MaxWidth: 3840  MaxHeight: 2160         Width: 3840     Height: 2160    BufSize: 0      ByFrm: 1        MCU: 1  Qfactor: 0      C2GEn: 0        DcfEn: 0

【调试信息分析】

记录JPEG 编码过程中，各通道的编码属性、状态。

【参数说明】

参数

描述

MODULEPARAM

OnePack

获取流的模式

0: 以多包模式获得流。

1: 以单包模式获得流。

JpegeMiniBufMode

分配数据流缓冲区的模式

0: 数据流缓冲区根据分辨率进行分配。

1: 数据流缓冲区大小的下限为 32KB。

用户需要确保分配的流缓冲区大小合适。

JpegClearStreamBuf

设置 JPEG 编码通道属性时是否清除流缓冲区。

0: 数据流缓冲区和上下文计数被保留。

1: 数据流缓冲区和上下文计数被清除。

JpegeDeringMode

启用 JPEG 编码通道的去环效果模式

0: 禁用

1: 启用

在相同的量化表和 Qfactor下，环现象可以减少，图像文件的大小也可以减小。但也会损失一些图像的清晰度和细节。

CHN ATTR

ID

通道号

bMjpeg

MJPEG encoding

No: JPEG 快照

Yes: MJPEG 编码

PicType

图像类型 YVU422 或 YVU420

MaxWidth

编码通道的最大宽度（单位：像素）

MaxHeight

编码通道的最大高度（单位：像素）

Width

图片宽度

Height

图片高度

BufSize

数据流缓冲区大小（单位：字节）

MCU

每个嵌入式 RISC-V 子系统 (ECS) 中的最小编码单元 (MCU) 数量

ByFrm

是否按帧获取数据流

取值: {0, 1}

Qfactor

通道 Qfactor

C2GEn

启用彩灰转换功能

取值: {0, 1}

DcfEn

JPEG 图像是否有缩略图

取值: {0, 1}
