【调试信息分析】

记录当前解码通道的使用状况及其属性配置。可用于检查属性配置以及当前解码通道统计状态。

【参数说明】

参数

描述

MODULE PARAM

VdecMaxChnNum

VDEC 支持的最大解码通道数

MiniBufMode

是否使用流缓冲区缩减模式

0: 禁用

1: 启用 (缩减模式仅在帧模式情况下有效)

enVdecVBSource

分配 VDEC 视频缓冲区的模式 (VB)

1: module VB

2: private VB

3: user VB

ParallelMode

VDH 解码模式

0: 非平行模式

1: 平行模式

MaxPicWidth

视频解码支持的最大宽度

MaxPicHeight

视频解码支持的最大高度

MaxSliceNum

视频解码支持的最大切片数

VdhMsgNum

VDH 报文池数量

VdhBinSize

用于存储 VDH 解码二进制数据的缓冲区大小

VdhExtMemLevel

用于 VDH 解码的处理器外内存分配水平

MaxJpegeWidth

待解码图像的最大宽度

MaxJpegeHeight

待解码图像的最大高度

SupportProgressive

是否支持逐行扫描格式

0: no

1: yes

DynamicAllocate

支持逐行格式时的缓冲分配模式

0: 静态分配

1: 动态分配

CapStrategy

解码图像最大宽度和高度的能力策略

0: 基于模块的能力策略

1: 基于通道的能力策略

CHN COMM ATTR & PARAMS

ID

解码通道号

TYPE

解码通道类型

PT_H264

PT_MJPEG

PT_JPEG

MaxW

配置解码图像的最大宽度

MaxH

配置解码图像的最大高度

Width

解码图像的宽度

Height

解码图像的高度

Stride

解码图像的对齐宽度

PixelFormat

图像的输出格式

PTS

解码的时间戳

PA

存储当前帧的地址

StrInputMode

VDEC 信道的数据流传输模式

模式可分为两种：

帧、流和兼容：分别以帧、流和兼容模式传输。

帧、流和兼容模式分别传输

BLOCK、NOBLOCK 和 TIMEOUT：流在

阻塞、非阻塞和超时模式下的数据流

StrBufSize

数据流缓冲区大小

FrmBufSize

帧缓冲区大小，仅在专用 VB 模式下有效

ParamPixelFormat

创建解码通道设定的帧格式，用于计算帧缓冲区所需大小

FrmBufCnt

帧缓冲区数量，仅在专用 VB 模式下有效

TmvBufSize

TMV 缓冲区大小。 该参数仅在私有 VB 模式下有效。

DispNum

显示帧数

取值范围: [0, 16]

DispMode

显示模式

取值范围: PLAYBACK 和 PREVIEW

SetUserPic

是否设置用户图像

EnUserPic

是否启用用户图像

Rotation

VDEC 图像的旋转角度

PicPoolId

帧缓冲区的 VB 池 ID，仅在私有 VB 和用户 VB 模式下有效。

TmvPoolId

Tmv 的 VB 池 ID，仅在私有 VB 和用户 VB 模式下有效。

STATE

VDEC 通道是否开始接收数据流

START（开始）： 通道开始接收数据流。

STOP（停止）： 通道停止接收数据流。

CHN VIDEO ATTR & PARAMS

ID

解码通道号

VfmwID

视频固件（VFMW）通道 ID

RefNum

参照帧数量

取值范围: [0, 16]

TemporalMvp

是否支持时域运动矢量预测

ErrThr

流误码率阈值

DecMode

解码模式

OutPutOrder

解码图像的输出序列

Compress

解码输出图像是否可被压缩

VideoFormat

待解码图像的数据格式

MaxVPS

支持的 VPS 的最大数量，只有 H.265 解码有效

MaxSPS

支持的 SPS 数量上限

MaxPPS

支持的 PPS 数量上限

MaxSlice

支持的最大切片数

CHN PICTURE ATTR & PARAMS

ID

解码通道号

Alpha

ARGB 格式 JPEG 图像的全局 alpha 值

VDEC CHN PERFORMANCE

ID

解码通道号

No.SendStreamPerSec

每秒钟发送码流的帧数

No.DecFramePerSec

每秒钟完成解码的帧数

HwDecTime

硬件解码时间
